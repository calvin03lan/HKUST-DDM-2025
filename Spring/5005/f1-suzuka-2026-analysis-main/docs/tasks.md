基於前面的「四人平行工作法」與「簡報腳本大綱」，以下是 A、B、C、D 四人在下週三諮詢前必須達成的具體任務（Minimum Viable Product, MVP）： 

 

隊員 A (總導播與 AI 負責人)：建構「報告骨架與 AI 使用紀錄」 

核心目標： 讓教授知道你們不是隨便亂問 ChatGPT，而是有系統地使用 AI。 

下週三前必須產出： 

AI Workflow 截圖與紀錄： 整理你們是如何構思 ⁠ skill.md ⁠ 或使用 Prompt 的。例如：「我們請 AI 扮演 F1 策略分析師，並輸入了哪些限制條件」。 

腳本大綱 (Script Outline) 1.0 版： 將前面我們討論的 20 分鐘 Flow，寫成一份有具體論點的 Word/Doc 檔。 

1000 字 Group Report 的「草稿結構」： 不需要寫完，但要有標題。例如：(1) AI 工具選擇 (2) 數據獲取流程 (3) 分工與協作模式。 

諮詢提問清單（代表團隊發言）： 準備向教授提問，例如：「老師，我們在 1000 字報告中，如果著重描述『利用 AI 清洗賽事遙測數據的過程』，這樣的方向符合要求嗎？」 

 

隊員 B (數據探勘者)：跑通「數據抓取流程 (Data Pipeline)」 

核心目標： 證明你們有能力拿到真實的 2026 鈴鹿站數據，而不是紙上談兵。 

下週三前必須產出： 

成功執行的 Code 片段： 無論是使用 Python 的 ⁠ FastF1 ⁠ 庫還是其他 API，必須要有能跑通的程式碼。 

至少兩份 CSV/DataFrame 樣本： 

一份是 Telemetry (遙測) 數據樣本（包含 Lando 和 Oscar 的速度、油門、煞車、檔位等，不需要全場，哪怕只有前 5 圈的 Sector 1 也行）。 

一份是 Lap Time (單圈時間) 數據樣本（包含圈數、單圈時間、輪胎配方）。 

諮詢提問清單（針對技術瓶頸）： 例如：「老師，我們在使用 FastF1 抓取 2026 年最新數據時，發現某個欄位有缺失值 (NaN)，請問您建議我們用內插法補齊，還是直接忽略該圈？」 

 

隊員 C (策略精算師與視覺化)：產出「核心圖表原型 (Mock Charts)」 

核心目標： 證明你們知道如何把生硬的數據，轉化為有說服力的商業/學術圖表。 

下週三前必須產出： (注意：如果 B 的真實數據還沒準備好，C 必須用假數據 (Mock Data)* 先畫出來！)* 

輪胎退化曲線圖 (Tire Degradation Plot) 原型： 畫出一張折線圖，X 軸是圈數，Y 軸是時間，展示中性胎 (Medium) 隨圈數增加而變慢的趨勢。 

遙測對比圖 (Telemetry Comparison Plot) 原型： 畫出一張曲線圖，展示兩位車手在特定區段的速度對比。 

諮詢提問清單（針對分析邏輯）： 例如：「老師，我們打算用『簡單線性迴歸』來擬合這條輪胎退化曲線，並以此定義進站窗口 (Pit Window)，請問這個數學模型對這次專案來說足夠嚴謹嗎？」 

 

隊員 D (防禦與故事總結者)：建立「What-If 邏輯與質性支撐」 

核心目標： 證明你們的故事不是瞎掰的，而是有官方資訊與邏輯推演作為支撐。 

下週三前必須產出： 

What-If 模型的邏輯敘述： 如果沒有安全車，Oscar 到底能不能贏？D 需要寫出這個推算的「公式邏輯」（例如：Oscar 當時每圈比對手快 0.2 秒，剩下 30 圈，所以預期優勢是 6 秒... 等）。 

質性資料彙整 (Qualitative Data)： 將 McLaren 官方報告、車手訪談中關於「練習賽液壓故障」和「安全車時機抱怨」的引言整理出來，作為 B 和 C 數據圖表的輔助說明。 

諮詢提問清單（針對敘事邏輯）： 例如：「老師，我們在最後的結論中加入了『假設安全車沒出現』的 What-If 推演，請問這種基於現有數據的『預測性模擬』，是否超出本專案的範圍？」
