# F1 鈴鹿 2026 分析

**研究問題：** 如果沒有安全車，Oscar Piastri 能贏嗎？

## 模擬器工作原理

### 5 分鐘看懂核心邏輯

想像 Piastri 正在跑鈴鹿賽道。我們不模擬複雜的 3D 物理，而是把賽道拉直成一條線：

1. **切段**：把鈴鹿賽道切成 28 段。例如：大直道 520m、S 彎群（大致對應 1–7 號彎）各段 20–250m、
   130R 高速彎區段、髮夾彎區段等。每段有一個類型（直道 / 高速彎 / 中速彎 / 低速彎）。

2. **彎道限速**：每個彎道有一個「最高過彎速度」(v_ref)。
   高速彎（大致對應 130R）約 200 km/h，低速彎（大致對應髮夾彎）僅 73 km/h。

3. **直道衝刺**：在直道上，車手盡全力加速。低速時加速度大（16.0 m/s²），
   高速時因空氣阻力加速度變小（6.2 m/s²），直到被極速 317 km/h 限制，
   或者必須開始煞車（17.4 m/s²）以符合下一個彎的限速。
   **注意**：這些數字是整圈校準後的「等效參數」，比原始遙測中位數更準確。

4. **每段時間 = 加速 + 巡航 + 煞車**：
   直道先加速到極速、定速巡航、再煞車進彎；
   彎道反過來，先煞車到彎心速度、定速過彎、再加速出彎。

5. **一圈時間**：把 28 段的通過時間加起來。

6. **輪胎退化**：每多跑一圈，輪胎磨損讓所有彎心速度微幅下降
   → 過彎變慢 → 圈速微增。HARD 胎退化極小（53 圈僅慢 ~0.3 秒），
   MEDIUM 退化較明顯。

7. **進站**：在進站圈加上 ~22 秒懲罰（進 pit lane + 換胎），並重設輪胎狀態。
   PIA 在第 18 圈進站（MEDIUM → HARD），ANT 在第 22 圈進站。

8. **重複 53 圈** → 得到整場比賽每一段、每一圈的速度與時間。

9. **分岔點模擬（Fork-Point）**：L1–L21 使用真實數據（零誤差），
   L22 開始模擬——這正是安全車改變一切的時刻。
   What-If 假設：ANT 在 L22 正常進站（+22 秒），而不是靠安全車免費進站。
   初始條件：PIA 累計 2017.2s（HARD，輪胎齡 3）、ANT 累計 2020.9s（HARD，輪胎齡 0）。
   PIA 在分岔點時領先 +3.7 秒。

10. **結論**：模擬結果顯示，如果沒有安全車，**PIA 以 +1.8 秒贏得比賽**。
    真實比賽中，安全車（L22–L27）讓 ANT 免費進站，抹平了差距並最終反超。
    交叉驗證（L28–L53 真實圈速 vs 模擬圈速）顯示 MAE 僅 0.3–0.4 秒/圈。

### 輸入

1. **賽道模型**：從結構化遙測 CSV 自動提取 28 個賽段（直道 + 高/中/低速彎）
2. **車輛模型**：4 個關鍵參數
   - `a_accel_low`：低速加速度（< 200 km/h）
   - `a_accel_high`：高速加速度（≥ 200 km/h）
   - `a_brake`：煞車減速度
   - `v_max`：極速

### 求解流程（技術細節）

1. **前向掃描**：從起點出發，計算每個邊界點能達到的最大速度
2. **後向掃描**：從終點回推，計算每個邊界點的煞車限速
3. **合併**：取兩者最小值 → 真實邊界速度
4. **計時**：對每段計算加速、巡航、煞車三階段用時

詳細數學推導請見 [docs/algorithm.md](docs/algorithm.md)。

### 輸出

`race_log` DataFrame（28 段 × 53 圈 × 車手數），包含：
- 每段的入彎/出彎速度、最低速度、通過時間
- 累計圈時與累計比賽時間
- 輪胎狀態與退化後彎心速度

Gap vs Lap 圖表：`outputs/gap_vs_lap.png`

## 敏感度分析

結論「PIA +1.8s」有多穩？我們測試了三個假設的變化：

- **ANT 進站時機**（L20–L24）：PIA 全部場景都贏
- **進站懲罰**（20–24s）：只有 ≤ 20s 才翻盤（真實值 ~22s）
- **輪胎退化率**：要高達 baseline 5 倍才翻盤（交叉驗證排除此可能）

## 規劃中：車手互動模型

目前兩位車手的模擬是完全獨立的（各跑各的，互不影響）。
敏感度分析顯示，gap 在所有合理場景中都 > 1.2 秒，不進入 DRS/互動範圍。
因此互動模型不會改變 What-If 結論，但保留作為未來擴展。

互動邏輯設計：

- **觸發條件**：當兩車在同一 segment 的累計時間差 < 1.5 秒
- **互動效果**：對該段的通過時間加擾動 `interaction_delta_s`
  - 前車防守壓力 → 可能微幅減速
  - 後車尾流效應 → 可能微幅加速
  - 四種結果組合：`(+,+)` 雙方搶位加速、`(-,-)` 互相阻擋、`(+,-)` 後車得利、`(-,+)` 前車得利
- **資料欄位**：`race_log` 已預留 `interaction` 和 `interaction_delta_s` 兩個欄位
- **對前端的意義**：當兩車距離 < 1.5 秒時，網頁上可能需要顯示「纏鬥中」的視覺提示

## 模型假設與限制

本模型是一個**一維質點模擬器**，以下是明確的簡化假設：

| 假設 | 說明 | 影響 |
|------|------|------|
| 質點模型 | 賽車無車寬，沿一維賽道運動 | 無法模擬超車路線選擇 |
| 忽略 DRS | 沒有顯式建模尾翼打開的加速效果 | 被校準的等效參數吸收 |
| 忽略油量變化 | 不考慮燃油減輕帶來的圈速提升（~0.05s/圈） | 被退化率部分抵消 |
| 忽略天氣/賽道演進 | 假設全場乾地、賽道抓地力不變 | 對鈴鹿乾地比賽合理 |
| 忽略車手互動 | 兩車手獨立模擬，無尾流/DRS/防守效應 | 敏感度分析證實 gap > 1.2s，不進入互動範圍 |
| 等效參數 | 校準後的加速度不是真實物理值，而是讓簡化模型產出正確圈速的等效值 | 見 [docs/qa.md](docs/qa.md) Q3 |
| 賽道模型來源單一 | 僅基於 PIA 最快圈的結構化遙測，非多圈平均 | 賽道特徵可能偏向最佳狀態 |
| 校準非唯一解 | 1 個目標（圈速）、3 個自由參數，理論上有無限組合 | 等比例縮放保持參數比例，見 Q4 |

詳細數學推導：[docs/algorithm.md](docs/algorithm.md) Section 1

## 隊友需要什麼

**模型已自動校準**，不需要隊友手動計算參數。

- 如果要加入新車手，只需要該車手的**真實最快圈時間**（一個數字，FastF1 即可查到）
- 匯出的 CSV 檔案可直接使用，見 [sim/README.md](sim/README.md)

### 給做可視化的隊友

| 檔案 | 行數 | 用途 |
|------|------|------|
| `outputs/gap_summary.csv` | ~636 行 | 每圈秒差，含 baseline + 11 個敏感度場景（用 `scenario` 欄位 filter） |
| `outputs/sensitivity_results.csv` | 12 行 | 每場景一行摘要（最終秒差、贏家），畫 sensitivity bar chart |
| `outputs/race_log.csv` | 1792 行 | 每段遙測（baseline），計算賽車在賽道上的位置 |
| `outputs/track_xy.csv` | 707 行 | 鈴鹿賽道中心線 (distance, x, y)，做 2D 賽道動畫 |

使用方式：
- 用 `gap_summary.csv` 的 `scenario` 欄位選擇不同場景，前端可以做滑桿切換
- 用 `race_log.csv` 的 `d_start` 映射到 `track_xy.csv` 的 (x, y) 座標，即可在 2D 賽道上顯示賽車位置
