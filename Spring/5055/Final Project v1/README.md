# Quick Draw Sketch Recognition

## Overview

This repository contains a deep learning pipeline for classifying hand-drawn sketches from the Google Quick Draw dataset. The model uses a **1D-ResNet + Bidirectional GRU + Attention** architecture to achieve **partial sketch recognition** — correctly identifying a drawing at multiple completion ratios (10%–100%) before it is finished.

The pipeline implements a **Heavyweight Specialist** training strategy: the model trains exclusively on sketches that are at least 40% complete and is evaluated on early sketches (10%, 20%, 30%) as a **zero-shot / out-of-distribution test** to measure robustness under severe domain shift.

### Categories (19)

| Group | Categories |
|-------|-----------|
| Shapes | circle, sun, donut, wheel, clock |
| Objects | umbrella, lollipop, hammer, toothbrush, spoon |
| Fruit | apple, pear, banana, strawberry |
| Animals | cat, dog, bear, mouse, rabbit |

## Data Pipeline & Reproduction

### Important: Raw Data Not Included

To save space (~1.5 GB), the raw `.ndjson` files are not included. You must regenerate the dataset locally.

### Step 1: Download Raw Quick Draw Data

Download the simplified `.ndjson` format for our 19 categories from the [Google Quick Draw Dataset](https://quickdraw.withgoogle.com/data).

**Command Line Download (Linux/macOS):**

```bash
mkdir -p raw_data
cd raw_data

for category in apple banana bear cat circle clock dog donut hammer lollipop mouse pear rabbit spoon strawberry sun toothbrush umbrella wheel; do
    curl -o "${category}.ndjson.gz" "[https://storage.googleapis.com/quickdraw_dataset/full/simplified/$](https://storage.googleapis.com/quickdraw_dataset/full/simplified/$){category}.ndjson.gz"
    gunzip "${category}.ndjson.gz"
done
cd ..

```

### Step 2: Process Dataset

Once all 19 `.ndjson` files are in `raw_data/`, run:

```bash
python data_processing.py
```

**What this script does:**

1. **Delta Conversion:** Converts absolute `[x, y]` coordinates to relative offsets `[dx, dy, pen_lift]`.
2. **Truncation:** Truncates each drawing into **9 completion ratios** (10%, 20%, 30%, 40%, 50%, 60%, 75%, 90%, 100%) to enable partial-sketch recognition training.
3. **Padding:** Pads/truncates all sequences to a unified length of 199 using `[0, 0, 0]`.
4. **Splitting:** Performs a stratified train/val/test split (70% / 15% / 15%).
5. **Output:** Saves the final tensors to `quickdraw_processed.npz` (Total samples: 378,088).

## Model Architecture

The specialist model uses a **1D-ResNet + BiGRU + Attention** pipeline optimized for sequence modeling:

1. **1D-ResNet Feature Extractor:** 2–3 residual blocks (Conv1d → BN → GELU → Conv1d → BN) with 1×1 shortcuts for channel alignment. It utilizes a sequence downsampling factor of 4 to extract geometric primitives efficiently.
2. **Bidirectional GRU:** `hidden_size=512`, 2 layers, `dropout=0.3`. Processes downsampled sequences in both directions to capture temporal context.
3. **Masked Multi-Head Self-Attention:** 8 heads with dynamic `key_padding_mask` mapped to actual sequence lengths, ensuring padded `[0,0,0]` steps are strictly ignored.
4. **Deep Classification Head:** `Linear → GELU → Dropout(0.3) → Linear (19 classes)`.

## Training & Evaluation

### Train the Specialist Model

```bash
python train_quickdraw.py --batch-size 1024 --epochs 100
```

* **Specialist Filter:** The model trains **only** on sketches with `ratio >= 0.40`. This forces the model to specialize in recognizing nearly-complete drawings.
* **Optimization:** Uses **AdamW** with a **CosineAnnealingWarmRestarts** scheduler to aggressively kick the model out of local minima.
* **Expected Runtime:** ~1 hour on an RTX 4070.

### Plot Training Curves

```bash
python plot_metrics.py --csv checkpoints/metrics.csv --out checkpoints/metrics.png
```

This generates a 2×2 grid visualization showing training/validation curves for **loss**, **Top-1 accuracy**, **Top-3 accuracy**, and **macro-F1**.

### Evaluate Zero-Shot Performance

```bash
python eval_truncations.py --npz-path quickdraw_processed.npz --checkpoint checkpoints/best_model.pt
```

This script evaluates the trained model on the full test set (all 9 ratios) to measure zero-shot performance on the early (10%, 20%, 30%) sketches. It outputs 10 granular bar charts and a Top-1 accuracy degradation curve to the `checkpoints/` directory.

## Inference Guide: Frontend Integration

For integrating the trained model into a frontend live-drawing canvas application.

### Input Requirements

| Attribute | Details |
| --- | --- |
| **Shape** | `(batch_size, 199, 3)` — default batch_size=1 for live inference. |
| **Features** | `[dx, dy, pen_lift]` per time step. **Do not send absolute (x, y) coordinates.** |
| **dx, dy** | Relative offset from previous point (Δx, Δy). |
| **pen_lift** | Binary: 1 if new stroke starts, 0 if continuing a continuous line. |
| **Padding** | Pad short sequences with `[0, 0, 0]` up to exactly 199 steps. |
| **Length** | Pass actual number of drawn steps (before padding) so attention ignores zeros. |

### Step-by-Step Inference Boilerplate

```python
import torch
import numpy as np
from train_quickdraw import ConvGRUClassifier

# ============ Load Model ============
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
checkpoint = torch.load("checkpoints/best_model.pt", map_location=device)

model = ConvGRUClassifier(
    input_size=3,
    num_residual_blocks=2,
    hidden_size=512,
    gru_layers=2,
    dropout=0.3,
    num_classes=19,
    device=device
)
model.load_state_dict(checkpoint["model_state_dict"])
model.to(device)
model.eval()

# ============ Prepare Input ============
# Example: Frontend sends a list of (dx, dy, pen_lift) tuples
raw_strokes = [
    (0.0, 0.0, 1),      # First point (new stroke)
    (5.2, -3.1, 0),     # Continuation
    (2.8, 1.4, 0),      # Continuation
    (6.0, 0.0, 1),      # New stroke
    (-4.5, 3.2, 0),     # Continuation
]
num_steps = len(raw_strokes)

# Pad to 199
sequence = np.array(raw_strokes, dtype=np.float32) 
padded = np.zeros((199, 3), dtype=np.float32)
padded[:num_steps] = sequence

input_tensor = torch.from_numpy(padded).unsqueeze(0).to(device)  # Shape: (1, 199, 3)
length_tensor = torch.tensor([num_steps], dtype=torch.long).to(device)

# ============ Run Inference ============
with torch.no_grad():
    logits = model(input_tensor, length_tensor)  
    probs = torch.softmax(logits, dim=1)  

# ============ Get Top-3 Predictions ============
top3_probs, top3_indices = torch.topk(probs, k=3, dim=1)
categories = [
    "circle", "sun", "donut", "wheel", "clock",
    "umbrella", "lollipop", "hammer", "toothbrush", "spoon",
    "apple", "pear", "banana", "strawberry",
    "cat", "dog", "bear", "mouse", "rabbit"
]

for i in range(3):
    idx = top3_indices[0, i].item()
    prob = top3_probs[0, i].item()
    print(f"Top-{i+1}: {categories[idx]} ({prob:.1%})")

```

## Troubleshooting

| Issue | Solution |
| --- | --- |
| `CUDA out of memory` | Reduce batch size: `--batch-size 256` or `--batch-size 128` |
| `ModuleNotFoundError: No module named 'torch'` | Ensure virtual environment is activated and `pip install -r requirements.txt` was run |
| `RuntimeError: CUDA is not available` | GPU/CUDA not detected. Verify NVIDIA GPU driver and CUDA 12.4 installation. |
| `FileNotFoundError: raw_data/*.ndjson` | Run data reproduction steps above to download raw files |
| `KeyError: X_test, y_test, ratios_test` | Run `data_processing.py` to generate `quickdraw_processed.npz` |
